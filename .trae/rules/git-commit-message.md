## Git 提交信息规范

### 语言要求
- 提交信息以中文为主，描述变更内容应清晰易懂。

### 提交格式
```
<type>: <简短描述>

<详细说明（可选）>
```

### type 类型
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档变更
- `style`: 代码格式调整（不影响功能）
- `refactor`: 代码重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具/依赖变更

### 规则
1. 标题行不超过 50 个字符，以中文描述变更内容。
2. 标题行末尾不加句号。
3. 详细说明与标题行之间空一行。
4. 详细说明描述变更的原因和做了什么，而非如何做。
